## How to Contribute

We welcome contributions from everyone! Here's how you can help:

### Ways to Contribute

1. **Code Contributions**
   - Bug fixes
   - New features
   - Performance improvements
   - Code refactoring

2. **Documentation**
   - README improvements
   - Code comments
   - Tutorial creation
   - Translation

3. **Testing**
   - Bug reports
   - Feature testing
   - Security audits
   - Device compatibility testing

4. **Design**
   - UI/UX improvements
   - Icon design
   - Screenshot creation
   - Promotional materials

5. **Community**
   - Answer questions
   - Help other users
   - Share feedback
   - Spread the word

### Contribution Process

1. **Fork the Repository**
   ```bash
   git clone https://github.com/akshitharsola/Secure-Vault.git
   cd Secure-Vault
   git checkout -b feature/your-feature-name
   ```

2. **Make Your Changes**
   - Follow code style guidelines
   - Add tests for new features
   - Update documentation
   - Commit with clear messages

3. **Submit Pull Request**
   - Push to your fork
   - Open PR with detailed description
   - Link related issues
   - Wait for review

4. **Get Recognition**
   - Your name will appear in GitHub contributors
   - Significant contributions may be highlighted here
   - Thank you for making SecureVault better!

### Code Style Guidelines

- **Language**: Kotlin only
- **Style**: Follow official Kotlin conventions
- **Architecture**: Respect Clean Architecture patterns
- **Naming**: Descriptive, self-documenting names
- **Comments**: Only for complex logic
- **Testing**: Unit tests for business logic
- **Documentation**: Update README for user-facing changes

### Pull Request Template

```markdown
## Description
Brief description of changes

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Breaking change
- [ ] Documentation update

## Testing
How did you test your changes?

## Screenshots (if applicable)
Add screenshots for UI changes

## Checklist
- [ ] Code follows style guidelines
- [ ] Self-reviewed code
- [ ] Added tests
- [ ] Updated documentation
- [ ] No new warnings
```

## Code of Conduct

By contributing, you agree to:

1. **Be Respectful**: Treat everyone with respect and kindness
2. **Be Constructive**: Provide helpful feedback and suggestions
3. **Be Patient**: Remember everyone is learning
4. **Be Inclusive**: Welcome contributors of all backgrounds
5. **Follow Guidelines**: Adhere to contribution guidelines

### Unacceptable Behavior

- Harassment or discrimination
- Trolling or insulting comments
- Political or off-topic discussions
- Spam or self-promotion
- Any illegal content

**Violations may result in temporary or permanent ban.**

## Frequently Asked Questions

**Q: Do I need to sign a CLA (Contributor License Agreement)?**
A: No, contributions are under MIT License automatically.

**Q: Can I contribute if I'm a beginner?**
A: Yes! Look for "good first issue" labels for beginner-friendly tasks.

**Q: How long does PR review take?**
A: Usually 1-3 days, depending on complexity and maintainer availability.

**Q: Can I contribute if English isn't my first language?**
A: Absolutely! We appreciate all contributions. We can help with language if needed.

**Q: Will I get paid for contributions?**
A: This is an open-source volunteer project. No monetary compensation, but you get recognition and experience!

**Q: Can I use this project in my portfolio?**
A: Yes! Contributions to open-source projects are great for portfolios.
